I downloaded Tourney, now what?

Updated 01/03/2015

There are two ways to set up a tournament in Tourney.

The easies way is to create a .tourney file. This is a simple text file that describes what engines will be participating and what the rules of the tournament will be. For instructions on how to create a .tourney file, see the section below.

Next you will need to open Tourney. It should look similar to:

Tourney
Loading tourney: 'default.tourney'... Success.
Opening book: default.book'...
Looking for previously build book 'default.book'...
Loading opening book: 'default.book'...
Loading previous tourney data... Nothing to load.

The following tourneys are currently loaded:
--> 1. Default Tourney

To select a different tourney from the list, type its name or number.
To load a tourney not listed, type 'load [filename]'
To see this list again, type 'ls'
For a list of additional commands, type 'help'

Broadcasting http on port 8080
Default Tourney> _

To load your .tourney file just type: load [filename]. Where [filename] is the path and name of the .tourney file you created. After it has loaded, you can choose to run the tournament or host it for other computers to play in. To start playing, just type: play. If at any point you want to stop the tournament, type: stop and the tournament will stop after the next game. To instead host the tournament, type: host.

If you would like to watch the tournament, open your web browser and navigate to http://localhost:8080/

Alternatively, if you do not want to create a .tourney file, you can set up a tournament with commands:

  • new - Creates a new tournament. Type this in before any of the other commands, otherwise you will be modifying the default tournament settings.
  • addengine - Type this in to be prompted about adding an engine to the tournament. You will need to set a name of the engine, it can be what ever you want. You will also need to know the file path of the engine and the protocol that it uses.
  • timecontrol <moves> / <per milliseconds> : <bonus milliseconds> - Sets the number of moves, milliseconds and increment per time control. For example 40 moves in 2 minutes with a 1 second increment after each move would be: timecontrol 40/120000:1000.
For more commands you can type commands or help.

How do I make a new tournament file?

Updated 01/04/2015

Tourney comes with two sample .tourney files to use as examples. One for Windows and one for Linux/MacOSX/BSD. The windows version is called sample_win.tourney. Opening it you will see:

{
     "Event": "Sample Windows Tourney",
     "Site": "localhost",
     "Date": "??",
     "Engines": [
         {
         "Name": "Senpai",
         "Path": "C:\\engines\\senpai.exe",
         "Protocol": "UCI"
         },
         {
         "Name": "Crafty",
         "Path": "C:\\engines\\crafty.exe",
         "Protocol": "WINBOARD"
         }
     ],
     "TestSeats": 1,
     "Carousel": true,
     "Moves": 40,
     "Time": 1000,
     "BonusTime": 0,
     "Repeating": true,
     "Rounds": 10,
     "BookLocation": "default.book",
     "BookMoves": 8,
     "RandomBook": false,
     "BookMirroring": true
}

It is best to open one of the samples and then save it as something else. After that, you can modify the fields you desire.

Adding Engines

To add more engines you can copy and what is between the [ ] after "Engines". When dealing with Windows file paths, one nuance to be aware of is to use \\ instead of \ in filenames. On the *nix systems, you don't have to worry about this.

For example you may want to have something like:

     "Engines": [
         {
         "Name": "Senpai",
         "Path": "C:\\engines\\senpai.exe",
         "Protocol": "UCI"
         },
         {
         "Name": "Crafty",
         "Path": "C:\\engines\\crafty.exe",
         "Protocol": "WINBOARD"
         },
         {
         "Name": "Stockfish",
         "Path": "C:\\engines\\stockfish.exe",
         "Protocol": "UCI"
         },
         {
         "Name": "Fruit 2.1",
         "Path": "C:\\engines\\fruit21.exe",
         "Protocol": "UCI"
         }
     ],

Make sure to put a comma after each '}' if there is another engine after it. But do not include a comma after the very last '}'. As seen above.

Tourament Modes

The format of the tourmament is determined by:

  • TestSeats - Determines how many engines will be played in gauntlet mode. When "TestSeats":1 this is just normal Gauntlet mode and the first engine listed will be played against every other engine (how many times is determined by the "Rounds" field).
  • Carousel - Controls the order in which the games are played. When "Carousel":true then each matchup will be played one at a time rather than all of the rounds for that matchup at one time. For example, suppose we have Engines A, B and C playing 2 rounds. Then the A vs B matchup will occur, then A vs C, then A vs B again and finally A vs C again. Opposed to A vs B, A vs B, A vs C, A vs C.
  • Rounds - How many games are played per matchup. For example if a tournament is Engine A vs Engine B for 1 round, we would just have 1 game: A vs B. If rounds were to instead be set to 2 we would have: A vs B then B vs A. For "Rounds":3 we would have: A vs B, B vs A, A vs B. And so on.

Time Control

Time control is determined by:

  • Time - Time measured in milliseconds per time control.
  • BonusTime - Milliseconds added to a players time after each move.
  • Moves - Complete moves per time control (not to be confused with plies).
  • Repeating - When "Repeating":true then after the time control is fulfilled a new time control will begin. If "Repeating":false then when either the number of moves or time have been satisfied, the game will be over.

For example, 40/2 time control would be represented by:

     "Moves": 40,
     "Time": 120000,
     "BonusTime": 0,
     "Repeating": true,

Or, 40/10:1 time control would be represented by:

     "Moves": 40,
     "Time": 600000,
     "BonusTime": 1000,
     "Repeating": true,

Opening Book

  • BookLocation - The path of the file being used as the opening book. This can be a pgn file or tourney's .book file. For Windows users make sure to \\ opposed to just \ for file paths. For example: "BookLocation":"C:\\tourney\\openings.book". This is not a concern for *nix users.
  • BookMoves - How many complete moves will be played before leaving the book. Set "BookMoves":0 to not use the opening book.
  • RandomBook - The moves from the book will be chosen at random rather than in sequence.
  • BookMirroring - When "BookMirroring":true each book position used will be played for both sides of a matchup. For example, if we have A vs B with opening X, then when we have B vs A opening X will also be used. This can help ensure fair opening book play.

After you have created your .tourney file it can be loaded by typing load [filename]. To start played locally type play. Or you can host a tournament by instead typing host.

What happens when a tournament finishes?

Updated 01/06/2015

When a tournament is finished running, Tourney will output the results to the screen and save them as <filename>.txt. A pgn version will also be saved as <filename>.pgn (where <filename> is the name of the .tourney file).

The progress of the tournament is saved as <filename>.data. Every time you open the .tourney file for this tournament, it will keep track of where you left off. If you would like to start the tournament from scratch you can use the delete command after loading the .tourney file. Then either the play or host commands.

Does Tourney support command line arguments?

Updated 01/04/2015

You can start Tourney with three different command line arguments right now.

The arguments include:

  • -play <filename> will open a .tourney file and start playing it on the local machine. For example you may type:

    $ tourney -play "C:\testing.tourney"

  • -host <filename> will open a .tourney file and act as a host for other computers to connect and play games. For example you may type:

    $ tourney -host "C:\testing.tourney"

  • -connect <IP> connect to the specified IP that is hosting a tournament. For example, to connect to your own computer after launching another instance of Tourney with the -host parameter, you may type:

    $ tourney -connect 127.0.0.1

How do you watch a tournament over the web?

Updated 01/06/2015

When Tourney is open, it starts automatically starts web server service. By default it broadcasts on port 8080. This can be changed in the settings file: tourney.settings.

From your machine, you can navigate to http://localhost:8080 to view the tournament.

For others to view the tournament, they will need to know your WAN IP address. If you are unsure what your IP is, you can simply google search, "what is my ip." Once you have determined your WAN IP, anybody can navigate their web browser to http://<WAN IP HERE>:8080 to watch the tournament. Also, if you are behind a router then you will need to have port forwarding enabled. If you are not sure how to do this, you can find a guide at www.portforward.com.

How do I host a tournament over the web?

Updated 01/06/2015

Launch Tourney and open a .tourney file (see this guide if you are not sure how to use .tourney files). Now type host. By default you will be serving the tournament on port 9000. This can be modified in the tourney.settings file. If you would like to play in the tournament locally, open another copy of Tourney. Then type: connect.

Any computer on your LAN can also play. Tourney will need to be installed on any computer that wishes to participate. However, the game engines do not. You will need to know your LAN IP. From any client computer use the connect <LAN IP> command from within Tourney.

To allow participants over the web, you will need to know your WAN IP and have the appropriate ports forwarded through your router. Any client can then just use the connect <WAN IP> command from within Tourney.

If you are unsure what your IP is, you can simply google search, "what is my ip." For help setting up port forwarding visit www.portforward.com.

How do I participate in somebody else's tournament over the web?

Updated 01/06/2015

The host will need to provide you with their IP address. Then from within Tourney use the connect <IP> command. To follow the progress of the tournament you can navigate your web browser to http://<IP>:8080 (where <IP> is the IP address of the host).

How do I make my own opening book?

Updated 01/06/2015

Tourney can generate an opening book from a pgn. If you specify a pgn file in the "BookLocation": field of the .tourney file, a book will automatically be generated.

After generating a book, it will be saved as <filename>.book in the default books folder. You can change the default books folder in tourney.settings.

To generate a book from a pgn manually, you can use the

buildbook <PGN filename> <move#> <filters...>

command. Where <move#> is how many moves into each game the book will contain. <filters> is optional and allows for constraints based on the tags in the PGN.

For example, the default opening book that comes with tourney was built from the first 14 moves of a PGN containing 2.2 million games. This giant PGN was filtered so that only draws from 2700+ ELO players would be included in the book. The command used to do this was:

buildbook millionbase-2.22.pgn 14 Result =1/2-1/2 WhiteElo >2700 BlackElo >2700

Be careful to get the spacing right between the filters. There is a space before a =, >, < but not after.

A shout out and big thanks goes to Ed Schroder for providing millionbase-2.2.pgn on his website. His site can be found here.

How can I use Tourney to help tune my engine?

Updated 01/06/2015

The results of a tournament are stored in <filename>.txt. The formatting of this file is determined by a template. You can modify templates/results.txt to control this the formatting. Once the formatting is the way you need it, it is easy to feed the outputted results file into a tuning algorithm. The next step from there is to write a quick script to loop between running a tournament, running the tuning algorithm and then compiling.

Of course, while the tuning tournament is running you can remotely view its progress. You can also use several computers to speed up the process.

If you don't have several computers, you can enlist your friends and family to help you. On Windows machines, you can schedule a task to run only when the system is idle. That means you can just schedule:

tourney.exe -connect <YOUR IP HERE>

Obviously, a lot of other schedule arrangements can be made. Such as when people are sleeping or are at work. Similar scheduling can be done on Mac and Linux machines but it isn't quite as easy.
